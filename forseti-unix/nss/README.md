# libnss_forseti

A thin NSS (Name Service Switch) module that resolves `passwd` and `group`
entries from the local **forseti-unixd** daemon over its Unix socket.

## What it does

glibc dlopens this `.so` into every process that performs a name lookup. When a
program calls `getpwnam`, `getpwuid`, `getgrnam`, `getgrgid`, or enumerates with
`getpwent`/`getgrent`, glibc walks `nsswitch.conf` and — if `forseti` is listed —
calls into this module. The module forwards the request to the daemon socket and
maps the reply into the C structs glibc expects.

It talks **only** to the daemon socket (`FORSETI_UNIXD_SOCKET`, default
`/run/forseti/unixd.sock`). There is **no network access and no secret material**
in the `.so` itself — the daemon owns all of that.

## nsswitch.conf

List `forseti` after `files` so local accounts always win and Forseti supplies
the rest:

```
passwd: files forseti
group:  files forseti
```

On a miss (no daemon, timeout, or unknown name) the module returns an empty
authoritative answer for enumeration and `NotFound` for point lookups, so libc
falls through cleanly without wedging the caller.

## Thin by design

Linked deps are only `libnss`, `libc`, `forseti-unix-proto`, and
`forseti-unix-client` (the blocking socket client: timeouts + `MSG_NOSIGNAL` +
fail-open `None`). No tokio/reqwest/rusqlite/tracing — this code runs inside
setuid `sshd`/`sudo`.

The `_nss_forseti_*` C symbols (including `setpwent`/`getpwent_r`/`endpwent` and
the `ERANGE` buffer packing) are generated by `libnss`'s `libnss_passwd_hooks!`
/ `libnss_group_hooks!` macros — we only implement the 3-method trait per
database. None of that boilerplate is hand-rolled.

## soname

The built artifact's `DT_SONAME` is stamped to `libnss_forseti.so.2` (via
`build.rs`). The Guix package (Part E) installs the artifact under that name
regardless.

## Runtime verification

Verification needs a host with the module installed under
`/lib/libnss_forseti.so.2`, the `nsswitch.conf` lines above, and the daemon
running:

```
getent passwd <user>
getent group  <group>
```

That end-to-end check is deferred to a Guix VM (Part E / Layer 5). The unit
tests here cover only the pure proto -> libnss conversion logic.
